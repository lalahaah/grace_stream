[Grace Stream] MVP 출시를 위한 Flutter 기술 스택 상세1. 프레임워크 및 언어Framework: Flutter (Latest Stable Version)Language: Dart특징: 단일 코드베이스로 iOS와 Android 동시 대응 및 고성능 UI 렌더링.2. 상태 관리 (State Management)Riverpod: 플러터 생태계에서 가장 권장되는 상태 관리 라이브러리.컴파일 타임 안전성 확보.성경 데이터 읽기, 재생 상태 제어 등 복잡한 비즈니스 로직 분리에 최적.3. 백엔드 및 클라우드 (Backend as a Service)Firebase: 인프라 관리 비용 최소화를 위해 채택.Authentication: 구글/애플/이메일 간편 로그인.Cloud Firestore: 사용자 묵상 데이터, 플레이리스트, 성경 데이터 동기화 (Rule 1 준수).Cloud Functions: Gemini API 호출 시 API Key 보안을 위한 프록시 서버 역할.Firebase Analytics: 사용자 행동 분석 및 MVP 개선 지표 확보.4. 핵심 기능별 라이브러리 (Key Libraries)4.1. 음악 및 영상 (Worship Player)youtube_player_flutter: 유튜브 영상을 앱 내에 임베딩하여 재생.유튜브 정책 준수 및 API 연동의 핵심.just_audio: 백그라운드 재생 및 오디오 제어가 필요할 경우 사용 (유튜브 정책 범위 내에서 활용).4.2. AI 및 데이터 처리 (AI Engine)google_generative_ai: Gemini 2.5 Flash API 연동을 위한 공식 Dart 패키지.dio: 외부 API 통신 및 네트워크 요청 처리.4.3. 로컬 스토리지 (Local Storage)Hive 또는 Isar: 고성능 NoSQL 로컬 데이터베이스.인터넷 연결 없이도 성경 본문을 읽을 수 있도록 오프라인 캐싱 지원.shared_preferences: 간단한 설정 값(다크모드, 폰트 크기 등) 저장.5. UI/UX 디자인 요소Design System: Material 3 (M3) 기반 커스텀 테마.Icons: lucide_icons 패키지 활용 (기존 기획의 일관성 유지).Fonts: Noto Sans KR (가독성), 세리프 계열 폰트(성경 본문용).6. 인프라 및 배포 전략CI/CD: GitHub Actions (코드 푸시 시 자동 빌드 및 테스트).Distribution: * Android: Google Play Console (Internal/Closed Testing).iOS: TestFlight를 통한 베타 테스트.Monitoring: Sentry 또는 Firebase Crashlytics (앱 오류 실시간 추적).7. MVP 개발 아키텍처 (Recommended)Architecture: Clean Architecture + MVVM (Model-View-ViewModel).Data: API 통신 및 데이터 로딩 (Repository).Domain: 비즈니스 로직 및 엔티티 (Use Cases).Presentation: UI 및 상태 관리 (Providers & Screens).8. MVP 단계별 구현 범위 (Scope)단계주요 구현 사항Phase 1 (Core)성경 뷰어 + 유튜브 기본 플레이어 + Firebase AuthPhase 2 (AI)Gemini API 연결 (오늘의 말씀 해설 및 찬양 추천 로직)Phase 3 (Social)구절 북마크 + 간단한 묵상 기록 및 저장 기능Phase 4 (Final Check)디자인 폴리싱 및 스토어 심사 준비
